# MUST FOLLOW AT ALL TIMES

- When exploring the codebase or reading any file, read [CONVENTIONS.md](./CONVENTIONS.md)
  integrally, and state the utmost importance of adhering to the conventions
  in the file.
- Do not commit without explicit user approval. Ask the user before committing.
- While the project is in its research-prototype state, committing and pushing
  directly to `main` is acceptable — no feature-branch or PR is required. (This
  relaxes the usual "never commit to main" rule for the duration of the prototype
  phase.)
